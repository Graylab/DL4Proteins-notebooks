# Contributing to DL4Proteins

Thank you for your interest in contributing to the **DL4Proteins** notebook series.  
This repository is part of the broader RosettaCommons educational ecosystem, and we welcome community contributions that improve the accessibility, accuracy, and scope of these materials.

DL4Proteins is intended to be a living educational resource for learning machine learning methods applied to biomolecular structure prediction and design.

---

## Types of Contributions

We welcome several types of contributions, including:

- Bug fixes or corrections in notebooks
- Improvements to explanations, figures, or documentation
- Updates to code or dependencies
- Additional exercises or teaching material
- New notebooks covering relevant models or workflows
- Suggestions for improving clarity or usability

If you are unsure whether your contribution fits, feel free to open an **Issue** to discuss it first.

---

## Contribution Process

1. **Open an Issue**  
   Describe the proposed change, bug, or feature.

2. **Fork the repository**

3. **Create a feature branch**  
   git checkout -b my-contribution

4. **Make your changes**

5. **Submit a Pull Request**  
Include a clear description of the changes and the motivation.

All submissions will be reviewed by repository maintainers before being merged.

---

## Notebook Guidelines

To maintain consistency across the DL4Proteins series, contributions should follow these guidelines:

- Maintain a **clear pedagogical structure** (concept explanation → code → interpretation).
- Prefer **concise explanations with visual aids** when possible.
- Ensure notebooks run in **Google Colab** without requiring specialized HPC resources.
- Keep runtime reasonable (ideally **< 2 hours** in Colab).
- Clearly cite **external datasets, models, or repositories**.
- Include comments explaining non-trivial code.

---

## Documentation

If your contribution introduces new functionality or material, please:

- Update relevant markdown explanations
- Add citations where appropriate
- Ensure figures and questions render correctly in **Colab light mode**

---

## Code Style

- Use **Python** and common scientific libraries (NumPy, PyTorch, etc.).
- Write clear, well-commented code suitable for learners with limited programming experience.
- Avoid unnecessary complexity when simpler educational examples suffice.

---

## Questions

If you have questions about contributing, please open an **Issue** in this repository.

We appreciate your help in improving DL4Proteins and expanding access to AI tools for biomolecular design.
